# 2. What Are AI Agents?

A crash course for the non-technical executive.

---

## Software Is Simpler Than You Think

Every software product Ensign uses — PCC, Workday, Microsoft 365, payroll, financial systems — is built from the same three components:

### The Three Layers

| Layer | What It Is | Example |
|-------|-----------|---------|
| **Database** | Where data lives — clinical records, HR files, financial transactions | PCC stores resident assessments; Workday stores payroll |
| **API / Data Access** | Rules for reading and writing that data — a structured doorway | Workday's API lets authorized systems pull HR data; PCC data can be accessed directly |
| **User Interface** | Screens humans use to interact — buttons, forms, dashboards | The PCC screens nurses click through every shift |

**The key insight:** Ensign pays millions per year for access to its own data through someone else's interface. AI agents don't need the interface — they connect directly to the data through the API.

PCC doesn't own Ensign's clinical data. Ensign does. PCC owns the screens. Agents bypass the screens.

---

## What an AI Agent Actually Is

An AI agent is software that can **read data, make decisions, and take actions** — without a human clicking buttons in a user interface.

### A Concrete Example

**Today (Human + SaaS UI):**
A billing specialist logs into PCC → navigates to a resident's account → reviews the billing codes and charges → cross-references the clinical documentation in a different PCC module → identifies a coding error → submits a correction → logs the change → emails the supervisor. Then Workday gets the month-end journal entry. **Time: 45 minutes.**

**With an AI Agent:**
The agent reads the billing data and clinical documentation from PCC simultaneously → identifies the coding error → prepares the correction with full documentation → presents it to a human for one-click approval. **Time: 90 seconds.**

The agent doesn't guess. It reads the same data a human would — it just reads it faster, across every system simultaneously, and never takes a day off.

---

## Why Now? Three Things Changed

### 1. Language models got smart enough
AI systems like Claude and GPT can now reason about complex business tasks — reading a clinical assessment, understanding a billing code, interpreting a regulation. This was not possible two years ago.

### 2. The APIs already exist
Every modern enterprise system provides data access — through APIs, data replication, or direct database connections. These are not hacks or workarounds. They are the intended access mechanisms. The connections already exist. Agents use them.

### 3. The cost collapsed
A billing audit that takes a human **6-8 hours** costs approximately **$0.40** in agent compute. A psychotropic medication audit that takes **3-4 hours** costs approximately **$0.15** in agent compute.

At Ensign's scale of 330 facilities, the math is extraordinary: one billing audit per week across all facilities costs **$6,864/year** in agent compute vs. **$200,000+** in human labor.

---

## The Speed Difference

| Task | Human | Agent |
|------|:---:|:---:|
| PDPM billing audit (1 facility) | 6-8 hours | 15 minutes |
| Psychotropic medication review | 3-4 hours | 90 seconds |
| Survey readiness score (1 facility) | Half day | 5 minutes |
| Daily operational metrics (census, staffing, AR) | 2-3 hours compiling | Real-time dashboard |
| HR credential scan (all staff, all facilities) | 1 day manual | Continuous, real-time |
| New vendor approval + invoice review | 45-60 min per vendor | 3 minutes |
| M&A pipeline update (new listings, comps) | Days of research | Overnight refresh |
| SNF industry news alert + action items | Manual monitoring | Continuous scan, daily brief |

At 330 facilities, these savings compound into **thousands of human-hours redirected every week** — from screen-clicking to resident care, clinical oversight, and strategic decisions.

---

## Human Process vs. Agent Process

Every operational process at Ensign follows the same pattern today. Here is how it changes:

| | **Human Today** | **Agent Alternative** |
|---|---|---|
| **Tools** | PCC screens, Workday portal, Excel, email, phone | Direct data connections to all systems simultaneously |
| **Access** | One system at a time, separate logins, manual cross-referencing | All systems at once — PCC + Workday + M365 + CMS in a single query |
| **Reasoning** | Years of on-the-job experience, varies by individual, subject to fatigue and turnover | PhD-level reasoning across clinical, financial, regulatory, and operational domains — consistent, 24/7 |
| **Speed** | Minutes to hours per task, one facility at a time | Seconds per task, all 330 facilities simultaneously |
| **Output** | Reports assembled manually, shared via email | Pre-analyzed briefings with recommendations, approve in 10 seconds |
| **Coverage** | Business hours, limited by headcount | Continuous monitoring, unlimited scale |

**The human doesn't disappear.** The human moves from data assembly to decision-making. The agent does the clicking, cross-referencing, and pattern detection. The human makes the call.

---

## Why Cross-System Matters

**Workday AI** can only see Workday data.
**PCC AI** can only see PCC data.
**Microsoft AI** can only see email and documents.

None of them can close this loop:

A clinical event in **PCC** → triggers a billing code in **PCC** → flows as a month-end JE to **Workday** accounting → affects the budget in the **board report** → requires a follow-up in **Microsoft 365**

**Only an agent layer that connects all systems can see the full picture.** At Ensign's scale — 330 semi-autonomous facilities across 15 states — seeing everything simultaneously is not a luxury. It is the competitive advantage.

One agent detecting a billing pattern across 10 facilities pays for the entire platform in a single finding.

# 3. How AI Actually Learns

Why the right knowledge — programmed in — is everything.

---

## The Brain Analogy

A person who has never seen a financial statement cannot audit one. A nurse who has never studied pharmacology cannot identify a drug interaction. **If the knowledge was never put in, it doesn't come out.**

AI works the same way. A language model (like Claude or GPT) learns by training on massive amounts of text — medical literature, financial reports, regulatory filings, coding documentation. It learns patterns, relationships, and reasoning. But if the training data doesn't include SNF operations, CMS F-tags, PDPM optimization, or Workday GL coding — the AI doesn't know it.

### Generic AI vs. SNF-Specific AI

| Generic AI Knows | Generic AI Does NOT Know |
|---|---|
| Medical terminology, general accounting | Ensign's PCC workflows and documentation standards |
| Common regulations, basic billing | How MDS assessments translate to PDPM payment rates |
| Email writing, document summaries | CMS F-tag interpretation for Ensign's operating model |
| General healthcare concepts | How to read a census report and spot what matters |

AI models can be **fine-tuned** — trained on specialized data to deepen domain expertise. Think of it as residency training after medical school.

---

## Who Programs the AI Matters

A generic AI that can "help with billing" is useful for a blog post. An agent built by someone who understands PCC billing codes, Workday GL mappings, CMS reimbursement rules, and Ensign's denial patterns is **useful for running a $5 billion operation.**

<div class="amber-callout">
<strong>The key question isn't "Does AI work?"</strong> The question is: <strong>who is programming it with the right knowledge?</strong> A SaaS vendor uses generic industry data. An expert with 13 years inside SNF operations and 3.5 years deep in AI engineering programs it with knowledge that matches how your business actually runs.
</div>

---

## What Changes with the Right AI Expertise

Ensign's leadership team is exceptional at operating skilled nursing facilities — record occupancy, record financials, 23 consecutive years of dividend growth prove it. But AI is a different domain. The danger is not making the wrong decision — it's **not knowing what questions to ask.**

| What the Team Hears | What AI Expertise Reveals |
|---|---|
| "PCC is adding AI features" | PCC AI only sees patient/billing data — cannot connect to HR, accounting, or communications |
| "Workday will add AI" | Workday AI costs extra (Flex Credits) and only sees payroll/HR/accounting data |
| "We'll leverage vendor partnerships" | Vendors prioritize their revenue model, not your operations |
| "AI is risky for HIPAA" | In-VPC processing is MORE secure than SaaS — which Ensign already uses |
| "We need a committee first" | Competitors deploying now get an 18-36 month head start |

None of this is obvious without deep expertise in both AI architecture and SNF operations.

# 4. Your Vendors' AI vs. Building Your Own

An analytical comparison — not hostile, but clear.

---

## What PCC and Workday Are Actually Offering

Both vendors are adding AI features. These are real products. But understanding the scope and limits is essential before building an enterprise strategy around them.

### PointClickCare: "Illuminate"

PCC's AI brand is **Illuminate** — an add-on analytics layer built on PCC's clinical data. It includes:
- **Predictive analytics** — hospital readmission risk, fall risk, patient deterioration alerts
- **PDPM optimization** — AI-driven MDS coding suggestions for reimbursement capture
- **Clinical documentation** — NLP-assisted note-taking and ambient listening features
- **Population health benchmarking** — cross-facility trend analysis across PCC's network

**Pricing:** Illuminate is an **add-on module** — not included in base PCC licensing. Industry estimates: $2-5 per bed per month on top of base EHR fees ($10-20+/bed/month). Ensign is already paying PCC millions annually, and AI costs extra.

**What PCC AI sees:** Patient records, MDS assessments, medication administration, billing/claims, census data. PCC handles all patient care and billing — but that is still only a portion of the data needed to run a SNF enterprise.

**What PCC AI does NOT see:** HR/payroll/scheduling, GL/accounting, email/calendar, contracts, supply chain, real estate, M&A data, board materials.

---

### Workday: "Illuminate" (Same Name, Different Product)

Workday launched its own "Illuminate" AI platform at Workday Rising (Sep 2024), expanded significantly through 2025-2026. It includes:
- **12 role-based AI agents** — recruiting, financial close, contract intelligence, employee sentiment, case management
- **Flex Credits** — a new consumption-based pricing model layered on top of existing subscriptions (~$0.10 per AI action, opaque pricing with no public rate card)
- **$1.1 billion Sana acquisition** (Nov 2025) — AI learning platform to make Workday "the front door for work"
- **Workday Data Cloud** — coming H1 2026 to blend external data (not yet available)

**The 75% adoption stat is misleading.** Workday counts any customer using any AI-embedded feature — including basic ML suggestions that have been in the product for years. The meaningful metric is the AI SKU attach rate: **only 35% of expansion deals** include paid AI capabilities.

**What Workday AI sees:** Payroll, HR, accounting/GL, benefits, recruiting data. **Zero patient data, zero CMS quality data, zero communications data.**

**What Workday AI cannot do:** Cross-reference staffing with patient outcomes. Correlate overtime costs with quality metrics. Monitor Five-Star ratings against workforce patterns.

---

## The Core Problem: Neither Sees the Other

<div class="amber-callout">
PCC AI sees patient and billing data. Workday AI sees payroll, HR, and month-end accounting. Neither sees the other. Neither can connect a patient event to a billing code to a GL entry to a board report. An independent agent layer sees all of it.
</div>

---

## Five Reasons Vendor AI Is Structurally Limited

**1. Vendor AI cannot cannibalize its own revenue.**
When Workday adds AI, it must not replace the per-seat licensing that generates its revenue. The AI feature is designed to justify the subscription — not replace it.

**2. Vendor roadmaps set your pace.**
Workday's AI roadmap is set by Workday's engineering priorities. Every 18-month product cycle is 18 months of competitive advantage handed to operators who deploy independently. Agentic deployment takes 30 days.

**3. The per-seat model is broken by agents.**
Salesforce has already moved to an all-you-can-eat agentic pricing model (AELA) because per-seat doesn't work when agents do the work. Workday's pricing model faces the same structural pressure.

**4. Data stays siloed.**
SaaS-vendor AI keeps data in each silo. Only a cross-system agent layer can connect patient events → billing → accounting → board reports.

**5. You get the same features as your competitors.**
Every SNF operator using PCC gets the same PCC AI features. There is zero competitive differentiation in vendor-provided AI.

---

## The Blockbuster Analogy

Blockbuster's fatal error was not incompetence. They had the capital, the customer relationships, and the content. Their fatal error was **trusting their existing business model to evolve fast enough.**

They asked their DVD distribution partners how to handle streaming. Netflix didn't.

Ensign asking PCC to build AI for clinical documentation is structurally identical: **asking the incumbent to cannibalize itself on your behalf.**

Blockbuster didn't fail because it lacked resources. It failed because it asked its existing partners to build its future.

# 5. How Software and SaaS Actually Work

What every executive should understand about the technology they pay for.

---

## The Three Layers — Why Agents Are Possible

Every software product — PCC, Workday, Salesforce — is built from three layers:

| Layer | What It Is | What Agents Need |
|-------|-----------|:---:|
| **Database** | Where data lives (Ensign's data, hosted by vendor) | Yes |

**The key insight:** Ensign pays per-seat licensing for the UI layer. Agents skip it entirely — they connect directly to the data through the same pathways the vendors' own products use. No hacking, no workaround.

Every system Ensign uses provides structured data access, documented by the vendors, covered by existing contracts, and already used by thousands of healthcare organizations.

**What's required to connect:**
1. PCC data access credentials
2. Workday tenant credentials
3. Azure AD app registration (for Microsoft 365)
4. AWS account access for in-VPC processing

The platform connecting all systems is already built. **30 days from credentials to production.**

## Governance Architecture

### Human-in-the-Loop
Agents propose. Humans approve. Every write operation to PCC, Workday, or any system shows a diff preview — exactly what will change, with full context and rationale. No autonomous writes without explicit human approval.

### Complete Audit Trail
Every action is logged: timestamp, user, data accessed, decision made, rationale provided. Three-layer audit architecture: hooks, middleware, database triggers. **SOX-ready from day one.**

### Role-Based Access
Agents inherit the user's permissions. A nurse sees her unit. A DON sees her facility. A regional director sees the region. An executive sees the enterprise. The same RBAC already enforced in Ensign's systems applies to the agents.

### Kill Switch
Any agent can be disabled instantly — per facility, per function, or enterprise-wide. No agent runs without authorization. Every agent can be stopped in seconds.

| Regulatory Requirement | How It's Addressed |

| **SOX** | Immutable audit logs, separation of duties, approval chains, financial data access controls |

## AI Safety Controls

- **Hallucination guard:** Every agent output cites its data sources and provides a confidence score
- **Read-only default:** Agents start in read-only mode. Write access requires explicit approval per operation
- **Governance levels:** Start with full human approval for every action → relax as trust is earned and validated
- **Confidence scoring:** 95%+ = recommend auto-approve. 80-94% = edge case, review recommended. Below 80% = requires human judgment

Your legal reviews the architecture. Your compliance sets the rules. Your people stay in control. The AI just moves faster between the guardrails.
